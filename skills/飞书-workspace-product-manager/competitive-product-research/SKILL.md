# 竞品调研（Competitive Product Research）

## 定位

对目标产品与竞品进行结构化对标分析，输出专业 HTML 可视化报告。主动搜集公开信息为主，用户材料为辅。**调研目标清晰即可启动，用户无需提供任何材料。**

## 快速启动

聚焦痛点（推荐）：

```text
我们APP的发帖转化率只有3%，想看看小红书和Instagram是怎么让用户顺畅发出第一条帖子的。
我方现状：右上角入口+空白编辑器+无草稿保存。
```

聚焦细分模块：

```text
调研长图文编辑器在弱网和意外退出时的「草稿保护与恢复机制」，对标知乎、公众号编辑器。
```

零门槛扫盲：

```text
帮我调研一下社区发帖功能的行业基建标配，对标小红书、Instagram、微博、知乎。
```

## 输入要素


| 要素   | 必填  | 说明                           |
| ---- | --- | ---------------------------- |
| 调研目标 | ✅   | 具体到一个可观察的功能节点 + 明确痛点/疑问      |
| 对标产品 | ✅   | 产品列表（≥2 个），建议同类型             |
| 调研动机 | 推荐  | 为什么调研？如"发帖转化率仅3%"            |
| 我方现状 | 推荐  | 产品名 + 当前做法 + 痛点（有锚点时针对性成倍提升） |
| 核心场景 | 可选  | 关键路径，不填则自动推断                 |
| 补充材料 | 可选  | 截图/录屏/链接/文档                  |
| 输出版本 | 可选  | 详细版（默认）/ 对外可分享版（自动脱敏）        |


## 合规边界

- ✅ 公开信息：应用商店、官网、公开评测/文章、用户评价、公开 API 文档
- ✅ 用户主动提供的材料：截图、录屏、Figma 导出、内部文档
- ❌ 禁止：进入第三方 App 抓取 / 绕过登录付费墙 / 批量爬取受限内容
- ✅ 对外分享版：自动脱敏人名/账号/内部链接/Key/Token/隐私数据

## 工作流

```
用户提交目标 → 信息确认 → 搜集+分析 → HTML 报告
```

### Phase 1 · 信息确认

收到输入后**不要直接生成报告**，先输出确认清单引导用户补充：

```
收到！在正式开始调研之前，我先帮你梳理关键信息。
已有的信息我帮你预填了，你可以逐项确认或修改，也可以直接说「确认，开始」。

━━━━━━━━━━━━━━━━━━━━━━
📋 信息确认清单
━━━━━━━━━━━━━━━━━━━━━━

1️⃣ 调研目标：[已提取 / 待补充]
2️⃣ 调研动机：[已提取 / 待补充]
3️⃣ 对标产品：[已提取 / 待补充]
4️⃣ 我方产品/现状：[已提取 / 待补充]
5️⃣ 核心场景：[已推断 / 待补充]
6️⃣ 关注维度：[已推断 / 待补充]
7️⃣ 输出版本：详细版（默认）/ 对外可分享版

⚠️ 聚焦度诊断：[通过 / 建议收窄——列出子课题选项]
💡 锚点状态：[有锚点→精准诊断 / 无锚点→行业深挖]

信息越详细报告越精准，也可以直接回复「确认，开始」。
```

确认清单处理规则：

1. **聚焦度诊断**：覆盖 ≥3 个独立子模块 = 过于宽泛，主动建议拆分为 2-3 个子课题让用户选择
2. **锚点检查**：未提供"我方现状"时主动询问——有锚点进入精准诊断模式，无锚点进入行业深挖模式
3. **推断标注**：推断的信息标注 `[推断]`，用户确认/跳过/只补部分均可
4. **部分补充合并优先级**：用户只补充部分字段时，按以下优先级合并——用户明确修改 > 用户确认的推断值 > 未提及但已预填的推断值（视为默认接受） > 未提及且无推断的字段（标注"未提供"并在报告中跳过或降级处理）

### Phase 2 · 搜集与分析

**信息来源**（按优先级）：📸 用户材料 > 🔗 公开信息（必须附 URL） > 💡 行业经验（标注"建议验证"）。所有来源统一编号 `SRC-xxx`，在报告来源索引登记。

**8 大分析维度**（D1/D2 不可裁剪，其余可按需标注 N/A + 原因）：


| #   | 维度     | 核心问题        | 检查要点                |
| --- | ------ | ----------- | ------------------- |
| D1  | 信息架构   | 能否快速找到目标功能？ | 入口层级、导航深度、可发现性      |
| D2  | 交互流程   | 操作链路是否顺畅？   | 步数、反馈速度、中断恢复、回退     |
| D3  | 视觉设计   | 视觉层级是否清晰？   | 信息密度、对比度、品牌一致性      |
| D4  | 文案策略   | 文案是否降低理解成本？ | 按钮文案、空态引导、错误信息      |
| D5  | 激励/增长  | 是否有效驱动用户行为？ | 新手引导、社交证明、进度可视化     |
| D6  | 异常/边界  | 边缘场景是否健壮？   | 空态、断网、权限拒绝、极端数据     |
| D7  | 跨端/跨主题 | 多端体验是否一致？   | iOS/Android 差异、暗黑模式 |
| D8  | 合规/无障碍 | 满足合规和可及性？   | 隐私提示、WCAG、屏幕阅读器     |


**场景拆解**：将核心场景拆为最小操作节点（每节点 = 意图 + 操作 + 反馈），逐产品×逐节点填写对标总览表。粒度示例和列规范详见 [playbook §1-2](references/research-playbook.md)。

### Phase 3 · 报告生成

严格按 [report-template.html](references/report-template.html) 生成 HTML 报告。模板中 `{{占位符}}` 为语义标记，生成时直接替换为实际内容输出纯 HTML，不保留占位符语法；`{{#列表}}...{{/列表}}` 表示循环区块，按实际数据条数重复生成对应 HTML 片段；`{{#条件}}...{{/条件}}` 表示条件区块，条件不满足时整块删除。

8 大分区：


| #   | 分区          | 内容要求                                 |
| --- | ----------- | ------------------------------------ |
| 1   | 封面          | 调研目标、对标产品列表、报告模式标签、**竞争力健康度徽章**、生成日期 |
| 2   | 摘要结论        | 3-5 条核心结论 + 最高优先行动项                  |
| 3   | 调研范围与信息来源   | 覆盖维度/端/主题 + 来源类型标注（📸🔗💡）           |
| 4   | 对标总览表       | 产品 × 场景节点的核心对比表（列规范见 playbook §2）    |
| 5   | 关键发现        | 按维度分组，每条含完整 6 要素                     |
| 6   | 可复用 Pattern | 命名 + 适用场景 + 交互/视觉规则 + 边界             |
| 7   | 落地建议        | P0/P1/P2 卡片，每条 6 字段齐全                |
| 8   | 验证计划 + 来源索引 | P0 建议的实验方案 + 成功标准 + 全部 SRC 映射表       |


## 评估标准

### 竞争力健康度（封面徽章）

**仅在有"我方锚点"时计算**（精准诊断/全景扫描模式）。无锚点模式（行业深挖/基建扫盲）不显示健康度。

**计算步骤**（确定性公式，禁止凭感觉）：

1. 为每条关键发现按严重度赋分：🟢 我方领先 = 100、⚪ 持平 = 70、🟡 明显差异 = 35、🔴 致命差距 = 0
2. D1/D2 维度的发现权重 ×1.5，D3-D8 权重 ×1.0
3. 健康度 = 所有发现的加权平均分（四舍五入取整）

**等级映射**：


| 分数     | 等级  | 标签    | 封面徽章色 |
| ------ | --- | ----- | ----- |
| 85-100 | S   | 竞争力领先 | 绿色    |
| 70-84  | A   | 竞争力健康 | 蓝色    |
| 55-69  | B   | 存在差距  | 黄色    |
| 0-54   | C   | 竞争力告急 | 红色    |


健康度评分展示在报告封面右上角，与报告模式标签并列。计算明细见 [playbook §9](references/research-playbook.md)。

### 严重度判定


| 等级      | 判定标准               | 量化参考               |
| ------- | ------------------ | ------------------ |
| 🔴 致命差距 | 核心路径上功能缺失或体验断裂     | 功能完全缺失，或步数差 ≥3     |
| 🟡 明显差异 | 体验/效率可感知差距，不阻断核心流程 | 功能都有但差距可感知，步数差 1-2 |
| 🟢 我方领先 | 我方优于 ≥2 个竞品，有证据支撑  | 可量化优势 + 证据引用       |
| ⚪ 持平    | 做法相近，无显著差距         | 用户无法感知差别           |


严重度须基于 [playbook 决策树](references/research-playbook.md)，不可主观臆断。

### 关键发现格式（6 要素缺一不可）

```
[维度标签]    — D1-D8
[发现标题]    — 一句话判断，≤25 字
[竞品做法]    — 2-3 句事实描述
[我方现状]    — 1-2 句
[差距分析]    — 本质原因 + 影响范围
[严重度+来源] — 🔴/🟡/🟢/⚪ + SRC-xxx
```

### Pattern 准入条件

全部满足才可提炼：≥2 个产品验证 + 场景可迁移 + 可用「适用场景 + 交互规则 + 视觉规则 + 边界条件」四元组完整描述。描述结构详见 [playbook §5](references/research-playbook.md)。

### 落地建议格式（6 字段缺一不可）


| 字段    | 要求                              |
| ----- | ------------------------------- |
| 优先级   | P0（当前迭代）/ P1（下一迭代）/ P2（Backlog） |
| 建议项   | 动词+宾语，≤30 字                     |
| 预期影响  | 量化或定性+置信度                       |
| 复杂度   | 低（≤1 周）/ 中（1-3 周）/ 高（>3 周）      |
| 依赖    | 前置条件                            |
| Owner | 产品/设计/前端/后端/算法                  |


**禁止模糊建议**——不接受"优化体验""参考竞品""后续跟进"等无具体动作的表述。反模式示例见 [playbook §6](references/research-playbook.md)。

## 质量自检

报告生成前逐项确认：

- 对标总览表覆盖所有「产品 × 核心场景节点」组合
- 每条关键发现 6 要素完整
- 每个 Pattern 满足准入条件
- 每条落地建议 6 字段齐全，无模糊表述
- 🔴 致命差距 ↔ P0 建议一一对应；P0 均有验证计划
- 所有 SRC 编号在来源索引中登记
- 💡 行业经验结论标注"建议实际验证"
- 无未标注来源的事实性结论
- 对标总览表单元格符合密度标准（做法 50-120 字，差异 30-80 字）
- 有锚点模式：竞争力健康度已按公式计算，等级与分数一致

## 注意事项

1. **先确认再开跑**：始终先输出信息确认清单，用户说"确认"才开始
2. **来源透明**：每条结论标注 SRC 编号，无来源的事实性结论禁止出现
3. **不编造数据**：不伪造竞品功能细节或虚构定量数据
4. **去 AI 化文风**：禁止空泛套话（"总的来说""值得注意的是"），结论必须具名、具数、具动作
5. **对外分享版**（`shareable`）自动脱敏，封面标注"对外可分享版"

## 自测用例概览


| 用例     | 场景                   | 核心通过标准                                  |
| ------ | -------------------- | --------------------------------------- |
| A 精准诊断 | 有我方锚点 + 聚焦目标 + 3 个竞品 | 确认清单输出→健康度徽章显示→对标总览表全覆盖→🔴↔P0 对应→验证计划完整 |
| B 行业深挖 | 无锚点 + 聚焦目标           | 总览表差异列自动切为"行业差异化价值"，无"与我方差异"列           |
| C 基建扫盲 | 无锚点 + 宽泛目标（≥3 模块）    | 先输出聚焦度提示→用户选"全部保留"→封面标签为"基建扫盲"          |
| D 最小输入 | 仅一句话 + 2 个产品名        | 先出确认清单，确认后基于推断生成完整报告                    |
| E 对外分享 | 标记 shareable         | 脱敏处理（无人名/账号/内部链接），封面标注"对外可分享版"          |


**失败判定**：漏分区 / 无 SRC 编号 / 发现缺要素 / 建议模糊 / 🔴 无对应 P0 / 宽泛目标未触发聚焦诊断 / 有锚点但无健康度徽章
