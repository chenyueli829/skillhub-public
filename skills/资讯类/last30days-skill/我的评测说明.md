根据 `last-30days-skill/SKILL.md` 里的说明，可以这样理解和「提问」：

## 这份技能在什么时候用

文档写明：当用户要 **「最近约 30 天内、多来源交叉印证的证据」** 时，才走 `last30days` 这条流水线。所以你的自然语言里最好明确包含：

- **时间**：例如「最近一周 / 最近一个月 / 2026 年以来」等（和「最近 30 天」一致即可）。
- **要什么**：事实、舆情、对比、推荐列表等，而不是泛泛聊天。

另外 YAML 里写了 `user-invocable: false`，表示这是 **给 Agent/运行时的内部规格**；你在 Cursor 里不必背命令行，用下面这种 **自然语言** 即可，由 Agent 按技能去跑脚本或等价流程。

## 怎么问（自然语言示例）

**1. 一般事实 / 舆情扫描**

- 「帮我查 **最近 30 天** 里，**{主题}** 在 Reddit、X、HN、YouTube 上大家主要在说什么，要有出处。」
- 「**{产品/公司}** 最近一个月社区反馈怎样？尽量多来源。」

**2. 对比（文档里要求输出结构）**

- 「**A 和 B** 最近一个月 **社区更站哪边**？各自优缺点，按技能里的对比结构写。」

**3. 「最好 / Top」类推荐**

- 「最近大家提得最多的 **{品类}** 工具/技能有哪些？要 **具体名字 + 大概被提到次数 + 来源**（不要空泛建议）。」

**4. 和某个人/品牌强相关时**

- 若话题对应 **可查的 X 账号**，可以说清楚实体名，便于 Agent 按文档去搜 handle 并加 `--x-handle=...`（你口头说「也看他们本人发帖」即可）。

**5. 控制深度（对应文档里的模式）**

- 要快：「**快速扫一眼** 最近 30 天就行。」
- 要均衡：「**正常深度** 查最近 30 天。」
- 要全：「主题很复杂 / 我要 **尽量不漏**，可以 **deep**。」
- 要机器可读：「结果给我 **JSON**（或给下游程序用）。」
- 限制来源：「**只要 Reddit + X**，不要泛泛网页。」

`argument-hint` 里的例子是 **`last30days codex vs claude code`**，对应命令行里传给 `last30days.py` 的核心查询串；你在对话里可以等价说成：「用 last30days 查 **Codex 和 Claude Code 最近 30 天社区对比**。」

## 提问时建议避免什么

技能里强调的 **反模式**：不要只给「Skills 很有用、控制在 500 行」这类 **脱离检索结果的通用建议**；要基于工具输出写 **具体名称、引用、互动数据** 等。

---

**一句话**：把你的需求说成 **「最近约 30 天 + 主题 + 要对比/要排名/要快或要深 + 要不要限定来源」**，就和这份 `SKILL.md` 的设计最一致。若你告诉我一个具体主题（例如某产品、某技术），我可以按上述结构帮你改成一条可直接复制给 Agent 的完整提问。
