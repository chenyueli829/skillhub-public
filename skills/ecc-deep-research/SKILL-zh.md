# 深度研究

利用 firecrawl 和 exa MCP 工具，从多个网络来源生成详尽、有引用的研究报告。

## 何时激活

- 用户要求深入研究某个主题
- 竞争分析、技术评估或市场规模估算
- 对公司、投资者或技术进行尽职调查
- 任何需要从多个来源综合信息的问题
- 用户提到"研究"、"深度调研"、"调查"或"目前……的现状是什么"

## MCP 依赖

至少需要以下之一：
- **firecrawl** — `firecrawl_search`、`firecrawl_scrape`、`firecrawl_crawl`
- **exa** — `web_search_exa`、`web_search_advanced_exa`、`crawling_exa`

两者同时使用可获得最佳覆盖。在 `~/.claude.json` 或 `~/.codex/config.toml` 中配置。

## 工作流程

### 第 1 步：理解目标

提出 1-2 个简短的澄清问题：
- "你的目标是什么——学习、决策还是撰写文章？"
- "有没有你特别关注的角度或深度？"

如果用户说"直接帮我查"——跳过此步，使用合理的默认设定。

### 第 2 步：规划研究

将主题拆分为 3-5 个研究子问题。示例：
- 主题："AI 对医疗的影响"
  - 当前医疗领域有哪些主要的 AI 应用？
  - 临床效果的衡量数据有哪些？
  - 监管方面有哪些挑战？
  - 哪些公司在这个领域领先？
  - 市场规模和增长趋势如何？

### 第 3 步：执行多源搜索

针对每个子问题，使用可用的 MCP 工具进行搜索：

**使用 firecrawl：**
```
firecrawl_search(query: "<子问题关键词>", limit: 8)
```

**使用 exa：**
```
web_search_exa(query: "<子问题关键词>", numResults: 8)
web_search_advanced_exa(query: "<关键词>", numResults: 5, startPublishedDate: "2025-01-01")
```

**搜索策略：**
- 每个子问题使用 2-3 种不同的关键词组合
- 混合通用搜索和新闻搜索
- 目标是获取 15-30 个不重复的来源
- 优先级：学术文献、官方来源、权威新闻 > 博客 > 论坛

### 第 4 步：深度阅读关键来源

对最有价值的 URL，抓取完整内容：

**使用 firecrawl：**
```
firecrawl_scrape(url: "<url>")
```

**使用 exa：**
```
crawling_exa(url: "<url>", tokensNum: 5000)
```

深度阅读 3-5 个关键来源。不要仅依赖搜索摘要。

### 第 5 步：综合分析并撰写报告

报告结构：

```markdown
# [主题]：研究报告
*生成时间：[日期] | 来源数：[N] | 置信度：[高/中/低]*

## 执行摘要
[3-5 句话的核心发现概述]

## 1. [第一个主要主题]
[带行内引用的研究发现]
- 关键要点（[来源名称](url)）
- 支撑数据（[来源名称](url)）

## 2. [第二个主要主题]
...

## 3. [第三个主要主题]
...

## 核心要点
- [可行动的洞察 1]
- [可行动的洞察 2]
- [可行动的洞察 3]

## 来源列表
1. [标题](url) — [一句话摘要]
2. ...

## 研究方法
通过网络和新闻搜索了 [N] 个查询。分析了 [M] 个来源。
调查的子问题：[列表]
```

### 第 6 步：交付

- **简短主题**：在对话中直接展示完整报告
- **长篇报告**：在对话中展示执行摘要和核心要点，完整报告保存到文件

## 使用子代理并行研究

对于广泛的主题，使用 Claude Code 的 Task 工具进行并行化：

```
并行启动 3 个研究代理：
1. 代理 1：研究子问题 1-2
2. 代理 2：研究子问题 3-4
3. 代理 3：研究子问题 5 + 交叉主题
```

每个代理独立搜索、阅读来源并返回发现。主会话将结果综合为最终报告。

## 质量规则

1. **每个论断都需要来源。** 不允许无来源的断言。
2. **交叉验证。** 如果只有一个来源提到，标记为"未经验证"。
3. **时效性很重要。** 优先选择最近 12 个月内的来源。
4. **承认信息缺口。** 如果某个子问题没有找到好的信息，如实说明。
5. **禁止幻觉。** 如果不确定，写"未找到充分数据"。
6. **区分事实与推断。** 明确标注估算、预测和观点。

## 示例

```
"研究核聚变能源的当前发展状况"
"深入对比 Rust 和 Go 在 2026 年后端服务中的表现"
"研究 SaaS 创业的最佳冷启动策略"
"美国房地产市场目前的情况如何？"
"调查 AI 代码编辑器的竞争格局"
```
