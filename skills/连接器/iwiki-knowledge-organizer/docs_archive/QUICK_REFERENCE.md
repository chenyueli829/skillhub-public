# iWiki Knowledge Organizer - 快速参考（历史文档，已按当前规则校正）

## 🚀 一分钟快速开始

### 基本使用

```
你: "拉取 https://iwiki.woa.com/p/1234567890"

AI 会询问（直接回车使用默认值）:
- 文档保存位置？(默认: 我的知识库/iwikis/)

说明：当前规范**不再使用** `我的知识库/images/` 与 `我的知识库/files/` 根目录；图片与附件统一放在与 `.md` **同级**的 `_images/`、`_files/` 下，详见 `SKILL.md`。

然后自动完成所有下载和整理工作！
```

---

## 📁 默认目录结构（当前规范）

```
我的知识库/
└── iwikis/                      ← 📄 Markdown 文档（按空间/文件夹/文档层级）
    └── {空间名}/{父路径}/
        ├── {文档A}.md
        ├── {文档B}.md
        ├── _images/             ← 🖼️ 同级共用图片目录
        │   ├── {文档A}/
        │   └── {文档B}/
        └── _files/              ← 📎 同级共用附件目录
            ├── {文档A}/
            └── {文档B}/
```

```
我的知识库/

│       └── 技术文档/          ← 文件夹
│           └── API设计.md     ← 文档
├── images/
│   └── FiT-资金中台/
│       └── 技术文档/
│           └── API设计/
│               ├── 1-架构图.png
│               └── 2-流程图.svg
└── files/
    └── FiT-资金中台/
        └── 技术文档/
            └── API设计/
                ├── 需求文档.pptx            ← PPT 附件
                ├── 数据模型.xlsx            ← Excel 附件
                ├── 技术方案.pdf             ← PDF 附件
                └── 1-腾讯文档-文档名称.md   ← 腾讯文档引用（编号-腾讯文档-文档名称）
```

## 🔧 配置参考

### 默认配置（v2.1）

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 文档目录 | `我的知识库/iwikis/` | Markdown 文档存储 |
| 图片目录 | `我的知识库/images/` | 图片资源存储 |
| 文件目录 | `我的知识库/files/` | 附件文件（PPT/Excel/Word/PDF等） |
| 图片命名 | `{序号}-{文件名}.{扩展名}` | 序号从1开始 |
| 存储模式 | 层级化 | 镜像 iwiki 结构 |

### 路径规则

```
文档: {保存位置}/{空间名}/{文件夹}/{文档名}.md
图片: {图片目录}/{空间名}/{文件夹}/{文档名}/{序号}-{图片名}.{扩展名}
附件: {文件目录}/{空间名}/{文件夹}/{文档名}/{附件文件名}.{扩展名}
腾讯文档: {文件目录}/{空间名}/{文件夹}/{文档名}/{编号}-腾讯文档-{文档名称}.md
（路径各段、文件/文件夹名均去空格，不添加其他符号）
```

```markdown

**链接**：[点击打开/下载](https://docs.qq.com/doc/DQVxxxxxx)
```

## 🎨 图片处理

### 序号命名优势

| 场景 | 示例 | 好处 |
|------|------|------|
| 快速定位 | "第一张图" | 直接找到 `1-*.png` |
| 保持顺序 | 文件排序 | 自动按数字排序 |
| 避免冲突 | 同名文件 | 序号区分 |
| 便于讨论 | "第三个流程图" | 清晰引用 |

### 示例对比

**之前**:
```
architecture-diagram.png
architecture-diagram-v2.png
architecture-diagram-final.png
```

**现在**:
```
1-architecture-diagram.png
2-architecture-diagram-v2.png
3-architecture-diagram-final.png
```

## ⚠️ 注意事项

### ✅ 推荐做法

1. 使用默认路径（`我的知识库/`）
2. 定期增量同步
3. 保留依赖清单
4. 不要手动修改文件结构

### ❌ 避免做法

1. 不要修改 `iwiki_id` 元数据
2. 不要手动重命名图片
3. 不要删除依赖清单
4. 不要混用旧版路径

## 🆘 常见问题

### Q: 图片显示不出来？

**A:** 检查路径
```bash
# 查看引用
grep "_images/" 文档.md

# 查看实际文件（以文档同级目录为准）
ls _images/文档名/
```

### Q: 如何自定义路径？

**A:** 在命令中指定
```
拉取 URL 到 自定义路径/
```

### Q: 支持哪些外部文件？

**A:** 
- ✅ 腾讯文档（docs.qq.com）
- ✅ 腾讯表格/幻灯片
- ✅ 微信文档
- ✅ Excel Online
