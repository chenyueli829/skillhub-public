## 📊 目录结构说明

### 文件与文件夹命名

- 所有文件、文件夹名称中的空格一律去掉，不添加连字符、下划线等其它符号。例：`0 整体视图` → `0整体视图`。

---

### 1. 根目录：`我的知识库/`

**作用**：统一的知识库根目录

**优势**：
- ✅ 所有 iwiki 相关资源集中管理
- ✅ 清晰的命名，易于识别
- ✅ 便于备份和迁移
- ✅ 避免文件混乱

---

### 2. 文档区：`我的知识库/iwikis/`

**作用**：存储从 iwiki 下载的所有文档

**层级结构**：
```
iwikis/
└── {空间名}/           ← 第1层：空间
    └── {文件夹}/       ← 第2层：文件夹（可多层）
        └── {文档}.md   ← 最终层：文档
```

**特点**：
- 📄 完全镜像 iwiki 的原始结构
- 📄 保持文档的组织关系
- 📄 支持多层级文件夹
- 📄 Markdown 格式，易于编辑

**文档格式示例**：
```markdown
---
iwiki_id: 1234567890
title: 系统架构设计
source: https://iwiki.woa.com/p/1234567890
creator: zhangsan(张三)
created: 2025-07-29 10:30:00
updated: 2026-02-14 15:45:00
parentid: 9876543210
external_dependencies: ../../../files/FiT-资金中台/技术文档/系统架构设计-dependencies.md
---

# 系统架构设计

## 概述
...
```

### 3. 图片区：同级 `_images/`

**作用**：存储文档中的所有图片资源（与 `.md` 同级目录共用一个 `_images/`）

**层级结构**：
```
{父路径}/
├── {文档名}.md
└── _images/{文档名}/
    ├── 1-图片名.png
    ├── 2-图片名.svg
    └── 3-图片名.jpg
```

**命名规则**：
- 🔢 **序号前缀**：从 1 开始，按文档顺序
- 📝 **原始文件名**：保留原始文件名（清理特殊字符）
- 📌 **格式**：`{序号}-{文件名}.{扩展名}`

**示例**：
```
FiT-资金中台/技术文档/
├── API设计.md
└── _images/API设计/
    ├── 1-接口架构图.png        ← 第1张图
    ├── 2-请求响应流程.svg       ← 第2张图
    ├── 3-数据模型.jpg          ← 第3张图
    └── 4-错误码说明.png         ← 第4张图
```

**Markdown 引用**：
```markdown
<!-- 在文档中引用图片 -->
![接口架构](_images/API设计/1-接口架构图.png)
```

---

### 4. 文件附件区：同级 `_files/`

**作用**：存储文档的附件文件和外部链接引用（与 `.md` 同级目录共用一个 `_files/`）

**层级结构**：
```
{父路径}/
├── {文档名}.md
└── _files/{文档名}/
    ├── 文件附件.pptx
    ├── 文件附件.xlsx
    └── 文件附件.pdf
```

**文件类型**：

**1. 可下载的附件**（实际文件）：
- 📊 **PPT/PowerPoint**: `.pptx`, `.ppt`
- 📈 **Excel 表格**: `.xlsx`, `.xls`
- 📄 **Word 文档**: `.docx`, `.doc`
- 📑 **PDF 文件**: `.pdf`
- 📦 **压缩包**: `.zip`, `.rar`, `.7z`
- 📝 **其他文件**: 各类可下载的附件

**2. 腾讯文档引用文件**（命名：`{编号}-腾讯文档-{文档名称}.md`）：
- 每个链接可单独一个文件；内容仅保留来源文档、链接（见 [TENCENT_DOCS_FORMAT.md](TENCENT_DOCS_FORMAT.md)）。

**示例结构**：
```
FiT-资金中台/技术文档/
├── API设计.md
└── _files/API设计/
    ├── 需求文档.pptx
    ├── 数据模型.xlsx
    ├── 1-腾讯文档-接口详细设计.md
    └── 2-腾讯文档-数据字典.md
```

**腾讯文档引用文件格式（简化）**：
```markdown
# 1-腾讯文档-接口详细设计

来源文档：[API设计](../../../iwikis/FiT-资金中台/技术文档/API设计.md)

**链接**：[点击打开/下载](https://docs.qq.com/doc/DQVxxxxxx)
```

## 🎯 路径计算规则

### 相对路径计算

文档在同级目录下引用 `_images/` 中的图片：

```
文档位置: 我的知识库/iwikis/FiT-资金中台/技术文档/API设计.md
图片位置: 我的知识库/iwikis/FiT-资金中台/技术文档/_images/API设计/1-架构图.png

计算步骤：
1. 从文档到"我的知识库"根目录：需要 3 个 ../
   - API设计.md → 技术文档 (../)
   - 技术文档 → FiT-资金中台 (../)
   - FiT-资金中台 → iwikis (../)
   - iwikis → 我的知识库 (../)  ← 3层

2. 从文档到图片（同级引用）：
   - 文档与 `_images/` 同级
   - 文档图片在 `_images/{文档名}/` 下

3. 完整相对路径：
   _images/API设计/1-架构图.png
```
