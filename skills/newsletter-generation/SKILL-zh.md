# 新闻通讯生成技能

## 概述

此技能生成专业、深度研究的新闻通讯，将来自多个来源的精选内容与原创分析和评论相结合。它遵循 Morning Brew、The Hustle、TLDR 和 Benedict Evans 等出版物的现代新闻通讯最佳实践，生成信息丰富、引人入胜且可操作的内容。

输出为完整的、可直接发布的 Markdown 格式新闻通讯，适用于邮件分发平台、网络发布或转换为 HTML。

## 核心能力

- 针对指定主题从多个网络来源研究和策展内容
- 生成主题聚焦或多主题的新闻通讯，保持一致的语调
- 撰写引人注目的标题、摘要和原创评论
- 以最佳可读性和浏览性构建内容结构
- 支持多种新闻通讯格式（每日摘要、每周综述、深度分析、行业简报）
- 包含相关链接、来源和引用说明
- 根据目标受众调整语调和风格（技术型、高管型、大众型）
- 生成具有一致品牌和结构的系列新闻通讯

## 何时使用此技能

**在以下情况下始终加载此技能：**

- 用户要求生成新闻通讯、邮件摘要或内容综述
- 用户请求某个主题的新闻或进展精选摘要
- 用户想创建定期的新闻通讯格式
- 用户要求将某个领域的最新进展汇编成简报
- 用户需要一份格式化的、可直接通过邮件发送的内容，包含多个精选条目
- 用户请求"每周综述"、"月度摘要"或"早间简报"

## 新闻通讯工作流程

### 第一阶段：规划

#### 步骤 1.1：了解新闻通讯需求

识别关键参数：

| 参数 | 描述 | 默认值 |
|-----------|-------------|---------|
| **主题** | 要覆盖的主要领域 | 必填 |
| **格式** | 每日摘要、每周综述、深度分析或行业简报 | 每周综述 |
| **目标受众** | 技术型、高管型、大众型或细分社群 | 大众型 |
| **语调** | 专业型、对话型、风趣型或分析型 | 对话-专业型 |
| **长度** | 短篇（5 分钟阅读）、中篇（10 分钟）、长篇（15 分钟以上） | 中篇 |
| **板块** | 内容板块的数量和类型 | 4-6 个板块 |
| **频率背景** | 一次性或定期系列的一部分 | 一次性 |

#### 步骤 1.2：定义新闻通讯结构

根据格式，选择合适的结构：

**每日摘要结构**：
```
1. 头条新闻（1 条，详细）
2. 快讯（3-5 条，简要）
3. 每日一数据 / 引言
4. 值得关注
```

**每周综述结构**：
```
1. 编者按 / 开场白
2. 头条新闻（2-3 条，详细）
3. 趋势与分析（1-2 条，原创评论）
4. 快讯（4-6 条，简要摘要）
5. 工具与资源（2-3 条）
6. 彩蛋 / 结尾
```

**深度分析结构**：
```
1. 引言与背景
2. 背景 / 为什么重要
3. 关键进展（详细分析）
4. 专家观点
5. 展望 / 影响
6. 延伸阅读
```

**行业简报结构**：
```
1. 执行摘要
2. 市场动态
3. 企业新闻与动向
4. 产品与技术更新
5. 监管与政策变化
6. 数据与指标
7. 展望
```

### 第二阶段：研究与策展

#### 步骤 2.1：多来源研究

使用网络搜索进行全面研究。**新闻通讯的质量直接取决于研究的质量和时效性。**

**搜索策略**：

```
# 当前新闻和进展
"[topic] news [current month] [current year]"
"[topic] latest developments"
"[topic] announcement this week"

# 趋势和分析
"[topic] trends [current year]"
"[topic] analysis expert opinion"
"[topic] industry report"

# 数据和统计
"[topic] statistics [current year]"
"[topic] market data latest"
"[topic] growth metrics"

# 工具和资源
"[topic] new tools [current year]"
"[topic] open source release"
"best [topic] resources [current year]"
```

> **重要**：始终检查 `<current_date>` 以确保搜索查询使用正确的时间上下文。切勿使用硬编码的年份。

#### 步骤 2.2：来源评估与筛选

评估每个来源并策展最佳内容：

| 标准 | 优先级 |
|-----------|----------|
| **时效性** | 优先选择最近 7-30 天的内容 |
| **权威性** | 优先使用原始来源、官方公告、知名出版物 |
| **独特性** | 选择提供新鲜视角或报道不足的故事 |
| **相关性** | 每个条目必须与新闻通讯的主题明确相关 |
| **可操作性** | 优先选择读者可以采取行动的内容（工具、见解、策略） |
| **多样性** | 混合新闻、分析、数据和实用资源 |

#### 步骤 2.3：深度内容提取

对于关键故事，使用 `web_fetch` 阅读完整文章并提取：

1. **核心事实** — 发生了什么、谁参与了、什么时候
2. **背景** — 为什么重要、背景信息
3. **数据点** — 具体的数字、指标或统计数据
4. **引述** — 相关的专家引述或官方声明
5. **影响** — 这对读者意味着什么

### 第三阶段：撰写

#### 步骤 3.1：新闻通讯头部

每份新闻通讯以一致的头部开始：

```markdown
# [新闻通讯名称]

*[标语或描述] — [日期]*

---

[可选：一句话预览内容]
```

#### 步骤 3.2：板块撰写指南

**头条新闻/精选条目**：
- **标题**：引人注目、清晰、面向受益（非标题党）
- **开头**：让读者关心的开场句（1-2 句）
- **正文**：关键事实和背景（2-4 段）
- **为什么重要**：与读者的世界建立联系（1 段）
- **来源链接**：始终标注并链接到原始来源

**快讯/简要条目**：
- **格式**：加粗标题 + 2-3 句摘要 + 来源链接
- **重点**：每个条目一个关键要点
- **效率**：读者无需点击即可获得核心信息

**分析/评论板块**：
- **声音**：新闻通讯对趋势或发展的独特视角
- **结构**：观察 → 背景 → 影响 → （可选）可操作的要点
- **证据**：每个声明都有数据或来源信息支持

#### 步骤 3.3：撰写标准

| 原则 | 实施 |
|-----------|---------------|
| **可浏览** | 使用标题、加粗文本、项目符号和短段落 |
| **引人入胜** | 以最有趣的角度开头，而非时间顺序 |
| **简洁** | 每句话都有存在的价值 — 无情地删除废话 |
| **准确** | 每个事实都有来源，每个数字都经过验证 |
| **标注来源** | 始终以内联链接的形式标注原始来源 |
| **有人情味** | 像一个有学识的朋友一样写作，而非新闻稿 |

**按受众调整语调**：

| 受众 | 语调 | 示例 |
|----------|------|---------|
| **技术型** | 精确，不解释行话，假设有专业知识 | "新 API 支持 gRPC streaming，通过 flow control windows 实现背压处理。" |
| **高管型** | 关注影响，直击要点，战略性 | "此次收购使 X 公司在企业市场获得 40% 的份额，直接威胁到在位者 Y 的定价能力。" |
| **大众型** | 易理解，使用类比，解释概念 | "可以把它想象成数据的万能翻译器 — 它让任何应用都能与任何数据库对话，无需学习新语言。" |

### 第四阶段：组装与润色

#### 步骤 4.1：组装新闻通讯

按照选定的结构模板将所有板块组合为最终文档。

#### 步骤 4.2：页脚

每份新闻通讯以以下内容结尾：

```markdown
---

*[新闻通讯名称] 是 [描述它是什么]。*
*[如何订阅/分享/反馈]*

*来源：所有链接均在文内提供。本新闻通讯策展并摘要公开信息，
附带原创评论。*
```

#### 步骤 4.3：质量检查清单

在最终确定之前，验证：

- [ ] **每个事实声明都有来源链接** — 无未注明来源的断言
- [ ] **所有链接均有效** — 来自搜索结果的已验证 URL
- [ ] **日期引用使用实际当前日期** — 无硬编码或假设的日期
- [ ] **内容是最新的** — 所有主要条目都在预期的时间范围内
- [ ] **无重复故事** — 每个条目只出现一次
- [ ] **格式一致** — 标题、项目符号、链接使用统一样式
- [ ] **覆盖平衡** — 不被单一来源或观点主导
- [ ] **长度适当** — 符合指定的长度目标
- [ ] **开头引人入胜** — 前 2 句话让读者想继续阅读
- [ ] **结尾清晰** — 新闻通讯以令人印象深刻或可操作的内容结束
- [ ] **已校对** — 无错别字、格式错误或不完整的句子

## 适配示例

### 科技新闻通讯
- 表情符号使用：✅ 适度（板块标题）
- 板块：头条新闻、深度分析、快讯、开源焦点、开发工具
- 语调：技术-对话型

### 商业/金融新闻通讯
- 表情符号使用：❌ 极少或不使用
- 板块：市场概览、交易动态、企业新闻、数据角、展望
- 语调：专业-分析型

### 行业专属新闻通讯
- 表情符号使用：适度
- 板块：监管更新、市场数据、创新观察、人事变动、活动
- 语调：专家-权威型

### 创意/营销新闻通讯
- 表情符号使用：✅ 自由使用
- 板块：案例聚焦、趋势观察、本周热点、好用工具、灵感
- 语调：热情-专业型

## 输出处理

生成后：

- 将新闻通讯保存到 `/mnt/user-data/outputs/newsletter-{topic}-{date}.md`
- 使用 `present_files` 工具向用户展示新闻通讯
- 提供调整板块、语调、长度或关注领域的选项
- 如果用户需要 HTML 输出，说明 Markdown 可以使用标准工具转换

## 注意事项

- 此技能与 `deep-research` 技能配合使用效果最佳，可实现全面的主题覆盖 — 对于需要深度分析的新闻通讯，请同时加载两者
- 始终使用 `<current_date>` 作为搜索和新闻通讯中日期引用的时间上下文
- 对于定期的新闻通讯，建议保持一致的结构，让读者形成预期
- 在策展时，质量胜过数量 — 5 个优秀的条目胜过 15 个平庸的
- 正确标注所有内容 — 新闻通讯通过透明的来源标注建立信任
- 避免摘要读者无法访问的付费墙内容
- 如果用户提供了特定的 URL 或文章要求包含，将它们与你策展的发现一起纳入
- 新闻通讯应在摘要中提供足够的价值，即使读者不点击每个链接也能受益
